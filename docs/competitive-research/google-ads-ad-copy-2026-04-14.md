# Google Ads Drafts for Vibers (2026-04-14)

Based on the SonarSource ad archive and insight memo, Vibers should avoid competing on broad "code quality", "SAST", and enterprise governance language. The stronger search angle is concrete founder/team pain:

- AI wrote the code, but nobody checked whether the flow actually works.
- Static analysis and AI comments do not equal a tested fix.
- Vibers sends a PR with fixes instead of only flagging issues.

## Messaging Rules

- Lead with outcome, not category: "PR with fixes" beats "code quality".
- Avoid overclaiming full security coverage. Stay honest: bug finding, flow testing, spec mismatch, hallucinated APIs.
- Prefer founder/team language over compliance language.
- Tie each ad group to the closest landing page. Do not send comparison intent to a generic home page if a comparison article exists.

## Suggested Acceptance Criteria

- Create first Google Ads search campaign with 3 primary ad groups:
  - AI Code Review
  - Human Code Review
  - Pre-Launch Bug Sweep
- Optionally add Competitor Intent as a separate low-budget campaign.
- Use phrase + exact match only for the first launch.
- Add the negative keyword list before campaign launch.
- Match each ad group to the correct landing page and keep message consistency.
- Review search terms after the first spend and prune broad enterprise/security traffic.
