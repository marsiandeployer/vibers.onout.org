# Vibers — Human-in-the-Loop Code Review for Vibecoded Projects

[![Star on GitHub](https://img.shields.io/github/stars/marsiandeployer/human-in-the-loop-review?style=social)](https://github.com/marsiandeployer/human-in-the-loop-review)

**Vibecoders ship code. We fix what's broken.**

Your developers use Cursor, Copilot, ChatGPT to write code fast. But who checks if it actually works, matches the spec, and doesn't have security holes?

Vibers reviews every commit, fixes issues, and sends pull requests with working code. Not comments — actual fixes.

## Quick Start (10 seconds)

**Option A — Tell your AI agent:**

```
Install skill from https://vibers.onout.org/SKILL.md
```

Your agent (Claude Code, Cursor, Codex) will set everything up automatically.

**Option B — Manual:**

1. Add [`marsiandeployer`](https://github.com/marsiandeployer) as collaborator to your repo
2. That's it. We start reviewing.

Spec/docs are optional. We figure out context from the code itself.

## What Gets Checked

| Category | What we catch |
|----------|--------------|
| **Spec compliance** | Feature doesn't match requirements, missing functionality |
| **Security** | Hardcoded secrets, SQL injection, XSS, missing auth checks |
| **AI hallucinations** | Non-existent APIs, deprecated methods, fabricated imports |
| **Logic bugs** | Edge cases, off-by-one, race conditions, null handling |
| **Visual/UI** | Broken layouts, wrong colors, mobile responsiveness |
| **Architecture** | Inconsistent patterns, copy-paste code, missing error handling |

## FAQ

**Do I need to share my spec?**
No. Spec is optional — we read the code and figure out context. But sharing a spec gives you better, more targeted reviews.

**What languages do you support?**
JavaScript/TypeScript, Python, React, Node.js, Django, Flask, Next.js, and most popular web frameworks.

**How fast is the review?**
Usually within 24 hours. Standard plan gets priority.

**What if I disagree with a fix?**
Comment on the PR. We'll discuss and adjust.

**Can I use this without GitHub?**
Yes — [write us on Telegram](https://t.me/onoutnoxon), send your code and spec.
